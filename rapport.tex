\section{Introduction}

Ce projet consiste a realiser une application de gestion de bibliotheque selon une architecture microservices.
L'application finale repose sur deux services independants, une interface web React, Docker, Docker Hub, PostgreSQL et Kubernetes.

Objectifs principaux :

\begin{itemize}[leftmargin=*]
  \item separer le domaine en deux microservices coherents ;
  \item mettre en place une communication REST entre services ;
  \item persister les donnees avec PostgreSQL ;
  \item conteneuriser chaque service ;
  \item publier les images sur Docker Hub ;
  \item deployer l'ensemble sur Kubernetes ;
  \item proposer une interface web de demonstration.
\end{itemize}

\section{Architecture generale}

Le projet se compose des elements suivants :

\begin{itemize}[leftmargin=*]
  \item \texttt{books-service} : gestion du catalogue de livres ;
  \item \texttt{user-loans-service} : gestion des emprunts ;
  \item \texttt{frontend} : interface React/Vite ;
  \item PostgreSQL pour chaque service ;
  \item ressources Kubernetes dans \texttt{k8s/base} et \texttt{k8s/loans}.
\end{itemize}

\section{Service A : Books Service}

\texttt{books-service} a ete developpe en Node.js avec Express.
Il gere les livres disponibles dans la bibliotheque.

Fonctionnalites :

\begin{itemize}[leftmargin=*]
  \item lister tous les livres ;
  \item recuperer un livre par identifiant ;
  \item creer un livre ;
  \item modifier un livre ;
  \item supprimer un livre ;
  \item filtrer les livres disponibles ;
  \item rechercher par titre ou auteur ;
  \item marquer un livre comme emprunte ;
  \item marquer un livre comme retourne.
\end{itemize}

Routes principales :

\begin{center}
\begin{tabular}{>{\ttfamily}p{4.5cm} p{8cm}}
\toprule
Route & Description \\
\midrule
GET /books & lister tous les livres \\
GET /books/:id & recuperer un livre \\
POST /books & ajouter un livre \\
PUT /books/:id & modifier un livre \\
PATCH /books/:id/availability & changer la disponibilite \\
PATCH /books/:id/borrow & enregistrer un emprunt cote catalogue \\
PATCH /books/:id/return & enregistrer un retour cote catalogue \\
DELETE /books/:id & supprimer un livre \\
\bottomrule
\end{tabular}
\end{center}

Le service calcule aussi un champ metier :
\texttt{AVAILABLE} ou \texttt{BORROWED}.

\section{Service B : User Loans Service}

\texttt{user-loans-service} a egalement ete developpe en Node.js avec Express.
Il prend en charge la logique metier des emprunts.

Fonctionnalites :

\begin{itemize}[leftmargin=*]
  \item creer un emprunt ;
  \item lister tous les emprunts ;
  \item lister les emprunts d'un utilisateur ;
  \item retourner un livre ;
  \item supprimer un emprunt ;
  \item verifier l'etat du service avec \texttt{/health}.
\end{itemize}

Routes principales :

\begin{center}
\begin{tabular}{>{\ttfamily}p{4.5cm} p{8cm}}
\toprule
Route & Description \\
\midrule
GET /health & verification du service \\
GET /loans & lister tous les emprunts \\
GET /loans/user/:userId & emprunts d'un utilisateur \\
POST /loans & creer un emprunt \\
PUT /loans/:id/return & retourner un livre \\
DELETE /loans/:id & supprimer un emprunt \\
\bottomrule
\end{tabular}
\end{center}

\section{Base de donnees}

Chaque service utilise PostgreSQL.

\subsection{Table books}

\begin{verbatim}
CREATE TABLE books (
  id VARCHAR(100) PRIMARY KEY,
  title VARCHAR(255) NOT NULL,
  author VARCHAR(255),
  available BOOLEAN DEFAULT true
);
\end{verbatim}

\subsection{Table loans}

\begin{verbatim}
CREATE TABLE loans (
  id SERIAL PRIMARY KEY,
  user_id VARCHAR(100) NOT NULL,
  book_id VARCHAR(100) NOT NULL,
  loan_date TIMESTAMP DEFAULT NOW(),
  return_date TIMESTAMP,
  status VARCHAR(20) DEFAULT 'active'
);
\end{verbatim}

\section{Communication inter-services}

La communication se fait entre \texttt{user-loans-service} et \texttt{books-service}.

Lors de la creation d'un emprunt :

\begin{enumerate}[leftmargin=*]
  \item le service des emprunts verifie que le livre existe ;
  \item il verifie que le livre est disponible ;
  \item il marque le livre comme emprunte ;
  \item il enregistre ensuite l'emprunt en base.
\end{enumerate}

Lors d'un retour :

\begin{enumerate}[leftmargin=*]
  \item l'emprunt est marque comme retourne ;
  \item le livre est remis en disponibilite dans \texttt{books-service}.
\end{enumerate}

Cette communication est la partie principale de l'integration metier entre les deux microservices.

Pour lancer toute l'application localement :

\begin{verbatim}

\end{verbatim}

\begin{itemize}[leftmargin=*]

  \item \texttt{user-loans-service} sur \texttt{http://localhost:3002} ;

\section{Conclusion}

Le projet final valide une architecture microservices complete :

\begin{itemize}[leftmargin=*]
  \item deux services independants ;
  \item deux bases PostgreSQL ;
  \item une communication REST metier ;
  \item une execution locale complete avec Docker Compose ;
  \item des deploiements Kubernetes distincts ;
  \item une interface web moderne pour la demonstration.
\end{itemize}

Le resultat final permet de gerer un catalogue, creer des emprunts, enregistrer les retours et visualiser l'etat du systeme dans une seule interface.
